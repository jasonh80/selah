# The Mega Workflow (owner-defined, 2026-07-20)

**Mega** is a designation a chapter EARNS: its text has been compared against
Jason's daily workup/rundown chats. Voice-generated ≠ Mega. Nothing is compiled
from partial workups and nothing is invented on Jason's behalf.

## The daily rhythm

1. Jason reads the chapter in his Bible.
2. Jason does his daily workup with ChatGPT — BEFORE reading the published
   chapter (uncontaminated eyes make the comparison honest).
3. The chapter launches through Studio self-serve (Jason approves prices +
   publish, as always).
4. Once a chapter has BOTH a finished workup and a live page, Codex runs the
   Mega comparison automatically: workup vs live chapter vs the Selah Voice.
5. Codex compiles the revision (editorial authority = copy gate; word-count
   flat; honesty rules senior). Claude builds the artifact + gates.
6. Applies are batched: one owner tap/script periodically, snapshot-first,
   rollback always.

## Standing jobs

- **Codex (ChatGPT side):** consistent workup template · log "workup complete:
  <chapter>" on board #29 · run the comparison when both halves exist.
- **Codex (repo side):** compile + exact-head reviews per the amended lane.
- **Claude:** artifacts, gates, batched apply script, live verification,
  Chapter Board upkeep.
- **Jason, forever:** the reading, the workup, prices, publish, the apply tap.
